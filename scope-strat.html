Testing Scope and Strategy

Testing Scope

The following system components and their related features are considered to be in scope for testing.

  • Extraction: The extraction process for retrieving the data from several different file formats and sources is a minimum requirement for the rest of the system to function correctly and requires testing from each point.
  • Transformation: The transformation process should be tested to ensure all data is transformed into the necessary format and all fields display true values.
  • Loading: The loading process requires to be tested to ensure that data is captured and stored in the correct location so that it can be used by the stakeholders.
  • Synchronous Pipes: Multiple events may trigger the ETL process at the same time, so testing synchronous pipes is required to ensure that the data is loaded appropriately in a timely manner.
  • Integrity: The integrity of the data that is loaded as the final product needs to be validated to ensure that the entire process works as expected and the data is usable and correct.
  • Error Handling: The system should be tested with data formats that are irregular and may not be recognized by the system to ensure that the errors are logged and handled appropriately.

The following system components are considered out of scope for testing:

  • User Interface: The user interface is out of scope since it was not included in the original design of the system
  • Security: Security and privacy testing is out of scope since the ETL system will be hosted on a platform that will regulate such protocols and testing
  • Reliability: uptime of the system is out of scope since the platform hosting the ETL system will assume responsibility for meeting all SLAs
