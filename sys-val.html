System and Validation Testing

System and validation testing are performed after all development work is completed and ensures that the entire system operates as expected. During validation testing, the whole system is tested to ensure that it functions as intended, so the entire ETL process will be verified end-to-end as the systemâ€™s stakeholders would expect it to operate. This ensures that the different modules of the system work cohesively together, and all dependencies were addressed and tested as a unit. In addition, with any ETL application or system, the integrity of the data is paramount. During the validation testing, the actual data that is loaded into the target system will be validated to ensure all transformations happened appropriately and in a time-efficient manner. In order to accomplish this, the following testing themes for validation testing have been identified:

  • ETL Processing / Synchronous Pipes
  • Data Integrity & Correctness
  • Error Handling

ETL Processing / Synchronous Pipes

Summary Test Procedure

The ETL Processing/Synchronous Pipes test procedures will focus on extracting data from one or many input data sources, transforming the data, and possibly denormalizing it, and then loading the data into a specialized format to an output destination such as a database, flat file or test file.

Data Integrity & Correctness

Summary Test Procedure

The Data Integrity and Correctness test procedures will focus on verifying whether valid and invalid conditions are accurately transformed based on the set rules defined in the Transform module. For example:

  • Able to convert string-based data to a date object.
  • Able to convert string-based Boolean to a true Boolean.
  • Able to convert a lookup key into a more complex object that contains the values that the key represents
  • Able to verify a postal address exists and is formatted correctly using a third-party verification service.
  • Capable of passing the transformed data to the Load Subsystem.

This procedure will also verify that the files to be transformed will adhere to the data types' requirements.

Error Handling

Summary Test Procedure

The Error Handling test procedure will focus on verifying whether:

  • Messages are generated for successful and failed notifications for every event within the Extraction, Transformation, and Load modules.
  • The system is capable of event reporting to the Syslog Server
  • Ensure that the Syslog Server is recording bad data and edge cases within the utility
