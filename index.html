Purpose

The purpose of this ETL Utility Test Plan is to outline and establish a minimum baseline for the testing efforts of this system. This plan contains the summary of the system's architecture, scope of the testing involved, a testing matrix to address the requirements, and a conclusion with outlined exit criteria. Additionally, this plan describes the types of testing involved, and is an iterative document that is modified as new requirements are outlined or discovered.

Scope

The scope of this test plan contains test cases for integration testing and system validation testing. This document provides an outline of the system as reference, including a summary architecture and requirements, as well as an outline of the test cases involved to meet these requirements and both test phases.

System Summary & Overview

The Extract-Transform-Load (ETL) utility is a web-based system that will allow data from a legacy database to be transformed into data used in modern systems that academic institutions can utilize for day to day tasks such as managing student records, online class delivery, and communicating with students. The utilityâ€™s design uses an event-driven architecture combined with a pipes-and-filters (primarily pipes) architecture and an object-oriented architecture. The architecture design contains three modules that will be tested against: Extract/Data-Entry Event, Transform, and Load.

The Extract/Data-Entry Event module watches for changes and dispatches an event when they occur. When the watcher notifies the utility of a change, the utility will then extract the new data from the specified source and package it for delivery to the Transformation Pipeline. The Transform module is the process of converting data entered from the legacy database and files into a standardized format and inserting the true values of the field. Lastly, the Load module will take the transformed data, collect load objects, and place the data into the respective destinations.
